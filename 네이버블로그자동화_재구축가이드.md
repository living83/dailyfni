# 네이버 블로그 자동 포스팅 시스템 재구축 가이드

## 1. 시스템 전체 흐름 요약

키워드 입력 → AI가 글 생성 → Playwright로 네이버 블로그에 자동 로그인 → 글 작성 → 발행

이 과정을 스케줄러가 매일 자동으로 실행한다.
발행 2시간 전에 글을 미리 생성해두고, 예약 시간이 되면 발행한다.


## 2. AI 에이전트 구성

총 4개의 에이전트가 순서대로 실행된다. 모두 Claude API를 호출한다.

### 2-1. Research Agent (상품 분석)

키워드를 받아서 상품 정보를 정리한다.

입력: 키워드 (예: "삼성 갤럭시 버즈3")
출력: JSON 형태로 아래 항목을 반환
- 상품명
- 카테고리
- 가격대
- 주요 특징 5개
- 장점 3개
- 단점 2개
- 타겟 고객층
- 활용 사례 3개
- 경쟁 제품 2개
- 핵심 차별점

정보가 부족하면 AI가 추론해서 채우고 "(추정)" 표시를 붙인다.

### 2-2. SEO Agent (검색 최적화)

Research Agent의 결과를 받아서 네이버 블로그에 최적화된 키워드와 제목을 만든다.

입력: Research Agent의 상품 분석 결과
출력: JSON 형태로 아래 항목을 반환
- 메인 키워드 1개
- 서브 키워드 5개
- 블로그 제목 3개 (친근체, 전문체, 초보체 각 1개)
- 태그 10개
- 카테고리 추천

제목 3개 모두 메인 키워드를 자연스럽게 포함해야 한다.
키워드 밀도 목표는 1.5~2.5%이다.

### 2-3. Writer Agent (글 작성) - 3개 톤으로 병렬 실행

Research + SEO 결과를 받아서 실제 블로그 글을 작성한다.
3가지 톤으로 동시에 생성한다.

톤 1 - 친근체:
- ~해요 말투
- 개인 경험담 포함
- 이모지 문단당 1~2개
- 2000~3000자

톤 2 - 전문체:
- ~입니다 말투
- 객관적 데이터 중심
- 이모지 없음
- 경쟁 제품 비교표 포함
- 별점 평가 포함

톤 3 - 초보체:
- ~이에요 말투
- 쉬운 용어, 비유 사용
- 전문 용어에 괄호로 설명 추가
- 2000~3000자

3개 톤 모두 글 마지막에 아래 문구를 필수로 넣는다:
"해당 글은 소정의 원고료를 받아 솔직하게 작성한 글입니다."

메인 키워드를 서론, 본문(3~5회), 결론에 자연스럽게 배치한다.

### 2-4. Reviewer Agent (품질 검수)

작성된 글을 검수한다. 3개 톤 각각에 대해 실행한다.

입력: 작성된 블로그 글
출력: JSON 형태로 아래 항목을 반환
- 맞춤법 검사: 오류 목록, 수정 제안, 점수
- SEO 검사: 키워드 배치 위치, 밀도 퍼센트, 상태(적정/과다/부족)
- 품질 검사: 글 길이, 소제목 수, 목록/표 사용, 중복 위험도, AI 생성 티 나는 정도
- 종합 점수: 맞춤법 25% + SEO 35% + 품질 40% 가중 평균
- 등급: A(85점 이상), B(70점 이상), C(55점 이상), D(55점 미만)
- 판정: 발행승인 / 조건부승인 / 수정필요

### 에이전트 실행 순서 정리

1단계: Research Agent 실행 → 상품 분석 결과 저장
2단계: SEO Agent 실행 (1단계 결과 입력) → SEO 데이터 저장
3단계: Writer Agent 3개 동시 실행 (1+2단계 결과 입력) → 글 3개 저장
4단계: Reviewer Agent 3개 동시 실행 (각 글 입력) → 검수 결과 저장

### 단순화 제안

실제로 3개 톤 중 1개만 발행하므로, 처음에는 이렇게 줄일 수 있다:
- Research + SEO를 하나로 합쳐서 Claude 1회 호출
- Writer는 1개 톤만 생성
- Reviewer는 생략하고 Writer 프롬프트를 잘 짜서 품질 확보

최소 구성: 키워드 → Claude 1회 호출(분석+글 생성 통합) → 발행


## 3. 네이버 로그인 방법

Playwright 브라우저 자동화를 사용한다. 2단계 전략이다.

### 3-1. 봇 탐지 우회 설정 (로그인 전에 반드시 필요)

브라우저 컨텍스트를 만들 때 아래 설정을 적용해야 한다.
이걸 안 하면 네이버가 자동화 브라우저로 감지해서 로그인이 안 된다.

적용할 설정 목록:
- navigator.webdriver 값을 false로 덮어씌운다
- Chromium 자동화 관련 플래그를 모두 비활성화한다
- User-Agent를 실제 Chrome 브라우저 값으로 설정한다
  예: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/134.0.0.0 Safari/537.36
- 타임존: Asia/Seoul
- 로케일: ko-KR
- 계정별로 프록시를 설정할 수 있다 (선택사항)

### 3-2. Step 1 - 쿠키 로그인 (빠른 경로)

한번 로그인에 성공하면 쿠키를 파일로 저장해둔다.
다음번에는 저장된 쿠키를 불러와서 로그인 없이 바로 접속한다.

절차:
1. 저장된 쿠키 파일을 불러온다 (경로: /data/cookies/account_{계정ID}.json)
2. 브라우저에 쿠키를 주입한다
3. 네이버 페이지에 접속한다
4. 페이지에 "로그인" 버튼이 있는지 확인한다
5. "로그인" 버튼이 없으면 → 이미 로그인된 상태 → 성공
6. "로그인" 버튼이 있으면 → 쿠키 만료됨 → Step 2로 진행
7. 성공 시 .apis.naver.com 도메인에도 쿠키를 공유한다 (댓글 iframe 호환용)

### 3-3. Step 2 - 아이디/비밀번호 로그인 (쿠키 실패 시)

쿠키 로그인이 실패하면 직접 아이디/비밀번호로 로그인한다.
최대 3회 재시도한다.

절차:
1. https://nid.naver.com/nidlogin.login 페이지에 접속한다
2. 아이디 입력 필드를 클릭한다
3. JavaScript로 아이디 값을 설정한다
   실행할 JS: document.querySelector("#id").value = "네이버아이디"
4. input 이벤트를 dispatch한다 (봇 탐지 우회용, 이걸 안 하면 입력이 안 된 걸로 처리됨)
5. 비밀번호 필드도 동일하게 JS로 값 설정 + input 이벤트 dispatch
6. 로그인 버튼을 클릭한다 (셀렉터: #log\.login 또는 .btn_login)
7. 3~5초 대기한다 (리다이렉트 시간)
8. CAPTCHA가 나타났는지 확인한다 (셀렉터: #captcha 또는 .captcha_wrap)
   - CAPTCHA가 나오면 자동 로그인 실패 → 수동 개입이 필요하다
9. URL에 "nidlogin"이 여전히 포함되어 있으면 로그인 실패 → 재시도
10. 로그인 성공 시 쿠키를 파일로 저장한다 → 다음번에는 Step 1으로 빠르게 로그인
11. .apis.naver.com 도메인에도 쿠키를 공유한다

중요: 아이디/비밀번호는 DB에 AES-256으로 암호화되어 저장되어 있다.
로그인할 때 복호화해서 사용한다. 암호화 키는 환경변수 MASTER_KEY (32자 이상)이다.


## 4. 블로그 글 발행 방법

로그인 성공 후 Playwright로 네이버 블로그 SE ONE 에디터에 글을 작성하고 발행한다.

### 4-1. 글쓰기 페이지로 이동

1. blog.naver.com에 접속한다
2. 글쓰기 버튼을 찾아서 클릭한다
3. 글쓰기 버튼을 못 찾으면 → 내 블로그(https://blog.naver.com/{네이버아이디})로 이동해서 다시 시도
4. 그래도 안 되면 → 직접 URL로 이동: PostWriteForm.naver

### 4-2. SE ONE 에디터 iframe 찾기 (핵심)

네이버 블로그 에디터는 iframe 안에 들어있다. iframe을 먼저 찾아야 글을 쓸 수 있다.

찾아야 할 iframe 이름들:
- mainFrame
- se_editFrame
- 또는 contenteditable="true" 속성이 있는 요소가 포함된 frame

iframe을 찾으면 그 안에서 에디터 요소를 조작한다.

### 4-3. 팝업/도움말 닫기

에디터가 열리면 도움말 팝업이나 임시저장 복구 다이얼로그가 뜰 수 있다.
이런 팝업을 자동으로 감지해서 닫는다.

### 4-4. 에디터 로딩 대기

에디터가 완전히 로드될 때까지 기다린다.
확인할 셀렉터: .se-component 또는 [contenteditable="true"]

### 4-5. 카테고리 선택

1. 카테고리 드롭다운을 연다
2. DB에 설정된 카테고리 이름을 검색한다
3. 일치하는 카테고리를 클릭한다

### 4-6. 제목 입력

에디터의 제목 필드를 찾아서 제목을 입력한다.

### 4-7. 이미지 삽입 + 본문 입력

본문을 ## (소제목) 기준으로 여러 섹션으로 분할한다.
각 섹션 사이에 이미지를 삽입해서 자연스러운 배치를 만든다.

순서:
1. 제목 아래에 대표 이미지 1장 삽입
2. 첫 번째 섹션 본문 입력
3. 이미지 삽입
4. 두 번째 섹션 본문 입력
5. 이미지 삽입
6. ... 반복

이미지는 Pillow로 생성한 키워드 이미지 또는 Gemini로 생성한 이미지를 사용한다.
광고글은 이미지 3장, 일반글은 이미지 1장이다.

### 4-8. 하단 링크 삽입 (광고글만)

광고글인 경우 본문 하단에 제휴 링크를 삽입한다.
일반글은 이 단계를 건너뛴다.

### 4-9. 태그 입력

태그 입력 영역을 찾아서 10개의 태그를 입력한다.

### 4-10. 발행

1. 발행 버튼을 클릭한다
2. URL이 PostView로 변경되는지 확인한다
3. URL이 변경되면 발행 성공
4. 발행된 URL을 DB에 저장한다


## 5. 스케줄링 (자동 실행)

APScheduler를 사용해서 매일 자동으로 실행한다. 3개의 작업이 있다.

### 작업 1: 기사 생성 (발행 시간 - 2시간)

예를 들어 발행 시간이 오전 10시면, 오전 8시에 글을 생성한다.

절차:
1. 스케줄러가 활성 상태인지 확인
2. 랜덤 휴식 체크: 10% 확률로 오늘 건너뛴다
3. 주말이면 30% 확률로만 실행한다
4. 활성 계정 목록을 가져온다
5. 각 계정의 티어에 따라 오늘 발행할 글 종류를 결정한다:
   - 티어 1 (신규): 매일 일반글만
   - 티어 2 (성장): 매일 일반글 + 5일마다 광고글
   - 티어 3 (전환): 2일마다 일반글 + 4일마다 광고글
   - 티어 4 (수익): 2일마다 일반글 + 3일마다 광고글
   - 티어 5 (마스터): 3일마다 일반글 + 3일마다 광고글
6. 키워드 큐에서 키워드를 가져온다
7. AI 파이프라인으로 글을 생성한다
8. 이미지를 생성한다
9. DB에 저장한다 (상태: generated)

### 작업 2: 발행 (예약 시간)

생성된 글을 네이버에 발행한다.

절차:
1. 상태가 articles_ready인 배치를 가져온다
2. 여러 계정이 있으면 라운드 로빈으로 순서를 섞는다 (같은 계정 연속 방지)
3. 각 글마다 2~4시간 간격을 둔다
4. 각 글에 대해:
   - 네이버 아이디/비밀번호를 복호화한다
   - 로그인한다
   - 글을 발행한다
   - 결과를 DB에 기록한다 (성공/실패, URL, 발행 시간)
5. 배치 상태를 업데이트한다 (all_success / partial_success / all_failed)
6. 연속 발행일 수를 업데이트한다

### 작업 3: 참여 활동 (선택사항)

다른 블로그 글에 좋아요와 AI 댓글을 남긴다.
계정의 활동성을 높이기 위한 용도이다.


## 6. 데이터베이스 주요 테이블

MySQL 8.0을 사용한다. 비동기 연결은 aiomysql이다.

accounts 테이블:
- id, account_name, naver_id(암호화), naver_password(암호화)
- account_tier (1~5), is_active
- proxy_server, proxy_username, proxy_password (선택)

keyword_queue 테이블:
- id, keyword, product_info, priority (ad/general)
- status (pending/used), last_used_at, next_available_at, used_count

publish_batches 테이블:
- id, keyword, post_type (ad/general)
- status (pending/articles_ready/publishing/all_success/partial_success/all_failed)
- success_count, failed_count

publish_history 테이블:
- id, batch_id, account_id, title, content, keywords(JSON)
- document_format (tutorial/review/analysis)
- status (generated/success/failed)
- naver_post_url, gemini_images(JSON), published_at

categories 테이블:
- id, account_id, category_name, is_default

scheduler_config 테이블 (1행만 존재):
- is_active, start_hour, start_minute
- days_of_week (1=월~7=일), min_interval_hours, max_interval_hours
- random_rest_enabled, weekend_low_prob
- consecutive_publish_days, last_publish_date
- footer_link, footer_link_text


## 7. 환경 변수

필수:
- ANTHROPIC_API_KEY: Claude API 키
- MASTER_KEY: AES-256 암호화 키 (32자 이상, 네이버 계정 암호화용)
- MYSQL_HOST, MYSQL_PORT, MYSQL_USER, MYSQL_PASSWORD, MYSQL_DATABASE

선택:
- ALLOWED_ORIGINS: CORS 허용 도메인 (쉼표 구분)
- GOOGLE_API_KEY: Gemini 이미지 생성용


## 8. 핵심 기술 포인트 정리

봇 탐지 우회:
- navigator.webdriver = false 필수
- 자동화 플래그 비활성화 필수
- 실제 Chrome User-Agent 필수
- 이걸 안 하면 로그인 자체가 안 됨

네이버 에디터 조작:
- SE ONE 에디터는 iframe 안에 있음
- iframe을 먼저 찾아야 에디터 요소 접근 가능
- iframe 이름: mainFrame 또는 se_editFrame

쿠키 관리:
- 로그인 성공 시 쿠키를 JSON 파일로 저장
- 다음 로그인 시 쿠키 먼저 시도 → CAPTCHA 회피
- .apis.naver.com에도 쿠키 공유 필요 (댓글 기능용)

계정 보호:
- 같은 계정으로 연속 발행하지 않음 (라운드 로빈)
- 발행 간격 2~4시간
- 10% 확률 랜덤 휴식
- 주말 발행 확률 30%
- 티어별 발행 빈도 차등 적용

아이디/비밀번호 입력 방식:
- 타이핑이 아니라 JS로 value를 직접 설정
- 반드시 input 이벤트를 dispatch해야 함
- 안 하면 네이버가 빈 값으로 인식함
